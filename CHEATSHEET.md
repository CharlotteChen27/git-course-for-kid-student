# Git 小抄

## commit

commit 是一次有名字的保存点。它和普通保存不一样，因为以后可以在 history 里看到它。

## diff

diff 是“找不同”。

- 绿色：新增的内容。
- 红色：删除或被替换的内容。

## branch

branch 是平行世界。你可以在分支里尝试新想法，不会马上影响 `main`。

## merge

merge 是把一个分支的成果带回现在站着的分支。

## conflict

conflict 通常是因为两个分支改了同一个地方。Git 不知道该听谁的，所以需要人来决定。

## push

push 是把本地 commit 上传到 GitHub。

## pull

pull 是把 GitHub 上的新 commit 拿回本地。
